## Key Design Decisions
- `companyId` is OPTIONAL (String?) — null means no filtering (backward compatible)
- When user has companyId set → only their company's data is returned
- When user has companyId null → all data is returned (no isolation)
- Auto-assignment on POST: if body doesn't specify companyId, uses the creating user's companyId
